# Contributing to QAMANIAI 🚀

First off, thank you for considering contributing to QAMANIAI! It's people like you that make QAMANIAI such a great tool.

## 🌈 How Can I Contribute?

### Reporting Bugs
*   Check the [Issues](https://github.com/qamanicode/qamaniai/issues) to see if the bug has already been reported.
*   If you can't find an open issue addressing the problem, open a new one. Be as detailed as possible.

### Suggesting Enhancements
*   Open a new issue and describe the feature you would like to see, why it would be useful, and how it might work.

### Pull Requests
1.  **Fork** the repository and clone it locally.
2.  **Create a Branch:** `git checkout -b feature/my-new-feature` or `git checkout -b fix/my-fix`.
3.  **Install Dependencies:** `npm install --force`.
4.  **Make Changes:** Write clean, documented, and type-safe TypeScript code.
5.  **Test:** Ensure your changes don't break existing functionality.
6.  **Commit:** `git commit -m "Add some feature"`.
7.  **Push:** `git push origin feature/my-new-feature`.
8.  **Open a Pull Request:** Explain your changes and wait for review!

## 📜 Coding Standards
*   Use **TypeScript** for all logic.
*   Follow [Inter](https://fonts.google.com/specimen/Inter) typography and [Tailwind CSS](https://tailwindcss.com/) patterns for UI components.
*   Ensure components are **responsive** and **accessible**.
